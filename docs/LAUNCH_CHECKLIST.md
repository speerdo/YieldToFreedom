# Yield to Freedom — Soft Launch Checklist

**Created:** 2026-06-26 · **Target:** Soft launch today
**Legend:** ✅ done/verified · 🔧 fixed this pass · ⚠️ needs your action (ops/manual) · ➖ deferred (post-launch)

This checklist was generated by auditing the codebase. Items marked ✅/🔧 were verified or fixed
in code. Items marked ⚠️ are operational steps that must happen outside the repo (Vercel dashboard,
DNS, Google Search Console, etc.) before or right after flipping the switch.

## 1. SEO Readiness

- ✅ **Per-page `<title>` + meta description** — `Seo.astro` requires both; every page passes them via `Base.astro`.
- ✅ **Canonical URLs** — `Seo.astro` emits `<link rel="canonical">` from `Astro.site` + pathname.
- ✅ **Open Graph tags** — title, description, url, type, image (1200×630), alt, site_name, locale.
- ✅ **Twitter Card** — `summary_large_image` with site handle `@yieldtofreedom`, title, desc, image, alt.
- ✅ **Structured data (JSON-LD)** — `schema` prop renders `application/ld+json` (home + ETF + blog pages).
- ✅ **Article OG metadata** — `og:type=article` + published/modified times wired for blog posts.
- ✅ **Sitemap** — `/sitemap.xml` dynamically includes static pages, all active ETFs (with `lastmod`), and blog posts. Falls back to static URLs if DB is unreachable at build/runtime.
- ✅ **robots.txt** — allows crawl, disallows `/api/` and `/app/`, points to sitemap.
- ✅ **RSS feed** — `/rss.xml` for the blog, linked in `<head>`.
- ✅ **noindex on transactional pages** — subscribe confirmed/invalid pages set `noindex`; 404 set `noindex` (🔧).
- ✅ **One `<h1>` per page** — verified across home, ETFs, ETF detail, blog, compare, stack-builder.
- ✅ **GA4 analytics** — conditionally injected when `PUBLIC_GA_MEASUREMENT_ID` is set (`Base.astro`).
- ⚠️ **Set `PUBLIC_GA_MEASUREMENT_ID` in Vercel** (Production) so analytics actually fire on the live site.
- ⚠️ **Submit sitemap to Google Search Console** + verify domain ownership after DNS is live.
- ⚠️ **Verify Bing Webmaster Tools** (optional but cheap reach).
- ➖ **Per-ETF / per-strategy meta description tuning** — current descriptions are serviceable; refine highest-traffic pages post-launch using Search Console queries.

## 3. Accessibility (a11y)

- ✅ **`<html lang="en">`** — set in `Base.astro`.
- ✅ **Landmark structure** — `<header>`, `<main>`, `<footer>` present site-wide.
- ✅ **Form labels** — contact form labeled; homepage newsletter uses `sr-only` label + `aria-live` status.
- ✅ **Decorative images hidden** — hero background image uses `alt=""` + `aria-hidden`.
- ✅ **Content images have alt** — blog/card images use `imageAlt` (falls back to title).
- ✅ **aria-labels on interactive controls** — theme toggle, mobile menu, footer/header nav, logo.
- ✅ **Visible focus states** — focus rings on inputs/buttons; skip-link reveals on focus.
- ⚠️ **Manual keyboard/contrast pass** — tab through home + ETF detail + a strategy page; verify all 4 themes (light/dark/tan/purple) meet contrast on key text. Quick to do, worth doing.

## 4. Performance / Lighthouse

- ✅ **Static output** — `output: 'static'`; pages prerendered to HTML (fast TTFB on Vercel CDN).
- ✅ **Non-blocking font load** — Google Fonts loaded via `media="print"` swap + `<noscript>` fallback.
- ✅ **Font preconnect** — `fonts.googleapis.com` + `fonts.gstatic.com`.
- 🔧 **Self-hosted + optimized imagery** — all 11 Unsplash photos downloaded, converted to right-sized **WebP** (`public/images/unsplash/`), references swapped site-wide. Hero is now **64 KB** (1920w). Removes the third-party dependency and the render-blocking cross-origin fetch. Script: `scripts/localize-unsplash.ts`.
- 🔧 **LCP image tuned** — hero now has explicit `width`/`height` (CLS) + `fetchpriority="high"` + `loading="eager"`; blog hero already `eager` + `fetchpriority="high"`.
- ✅ **Below-fold images lazy** — card/grid images use `loading="lazy"`.
- ✅ **CLS guarded** — image containers use fixed `aspect-ratio`; hero + blog hero have explicit width/height.
- ✅ **Vercel Web Analytics enabled** — `webAnalytics: { enabled: true }` in adapter.
- ⚠️ **Run Lighthouse on the deployed Preview/Prod URL** — localhost numbers are misleading. Target ≥90 across the board.

## 6. Security / Headers

- 🔧 **Security headers added** — `vercel.json` now sets `X-Content-Type-Options`, `X-Frame-Options: SAMEORIGIN`, `Referrer-Policy`, `Strict-Transport-Security` (HSTS, 2yr + preload), and `Permissions-Policy`.
- ✅ **Secrets server-side only** — FMP/Tiingo/Resend/Stripe/Clerk secret keys read from env, never `PUBLIC_`-prefixed.
- ✅ **Cron endpoints authenticated** — `/api/cron/*` guarded by `CRON_SECRET` bearer (`cron-auth.ts`).
- ✅ **API/app paths excluded from crawl** — robots disallows `/api/` and `/app/`.
- ✅ **No stray `console.log`** in `src/`.
- ⚠️ **Confirm all env vars set in Vercel Production** — `DATABASE_URL`, `TIINGO_API_KEY`, `CRON_SECRET`, `RESEND_API_KEY`, `RESEND_FROM`, `PUBLIC_SITE_URL`, `PUBLIC_GA_MEASUREMENT_ID` (and Clerk keys only if Phase-2 auth is enabled).
